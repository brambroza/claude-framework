# User Training Skill

## Training Package ที่ต้องสร้าง
1. Training Agenda
2. Slide Deck outline (ภาษาไทย)
3. Hands-on Exercise Sheet
4. Quick Reference Card (A4 1 หน้า)
5. Post-training FAQ

---

## Training Agenda Template

```markdown
# Training Agenda — [ชื่อระบบ]
วันที่: [วันที่] | เวลา: [เวลา] | สถานที่/Online: [ที่]
วิทยากร: [ชื่อ] | กลุ่มเป้าหมาย: [end user / admin]

---

| เวลา | หัวข้อ | รูปแบบ |
|---|---|---|
| 09:00-09:15 | แนะนำระบบและวัตถุประสงค์ | บรรยาย |
| 09:15-09:45 | Demo ภาพรวมระบบ | Demo |
| 09:45-10:30 | [Module A] — ฝึกปฏิบัติ | Hands-on |
| 10:30-10:45 | Break | |
| 10:45-11:30 | [Module B] — ฝึกปฏิบัติ | Hands-on |
| 11:30-12:00 | Q&A + สรุป | |

รวม: 3 ชั่วโมง
```

## Slide Deck Outline

```markdown
# [ชื่อระบบ] — คู่มือการใช้งาน Training

## Slide 1: ระบบนี้ช่วยอะไรคุณ
- ก่อน: [ทำอะไรอยู่ตอนนี้ — เชื่อมกับปัญหาเดิม]
- หลัง: [ชีวิตจะดีขึ้นยังไง]

## Slide 2-N: ทีละ Module
สำหรับแต่ละ module:
- ทำอะไรได้บ้าง (1 slide)
- Demo step-by-step (1-2 slides)
- ข้อควรระวัง (1 slide)

## Slide สุดท้าย: ถ้ามีปัญหา
- ติดต่อใคร
- Line ID / เบอร์
- เวลาทำการ
```

## Hands-on Exercise Sheet

```markdown
# แบบฝึกหัด — [ชื่อระบบ]

## Exercise 1: [งานพื้นฐานที่ทำบ่อยที่สุด]
**เป้าหมาย:** ฝึก[งาน]ได้ด้วยตัวเอง

**ขั้นตอน:**
1. Login ด้วย account: test@example.com / Test1234
2. ไปที่เมนู [ชื่อเมนู]
3. กด [ปุ่ม]
4. กรอกข้อมูลตามตัวอย่าง:
   - ชื่อ: ทดสอบ 001
   - [field อื่น]: [ค่า]
5. กด "บันทึก"
6. ตรวจสอบว่าเห็น [ผลลัพธ์]

**ผ่านเมื่อ:** เห็นข้อความ "[ข้อความ success]"

---

## Exercise 2: [งานที่สอง]
[เหมือนกัน]
```

## Tips การ Training SME ไทย
- ใช้ตัวอย่างที่เป็น business ของลูกค้าจริงๆ ไม่ใช่ "Foo/Bar"
- ทำ test account ที่มีข้อมูล realistic พร้อมแล้ว
- Record การ training ไว้ด้วยเสมอ
- ให้ print Quick Reference Card ไว้ที่โต๊ะ
- นัด follow-up call 2 สัปดาห์หลัง go-live
